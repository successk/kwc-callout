<kwc-bootstrap> Documentation

Properties

Attribute Type Default Properties Required Description
header String null - × The callout header
show Boolean true reflectToAttribute + notify × This component is shown. You can use a variable to change and listen to change of the value.
closeable Boolean false - × If true, a close button will be shown
autoclose Number 0 - × If set and different from 0, the callout will be closed after {autoclose}ms. The component will not be destroyed, if you want to do so, you will need to do it yourself.

From v0.x to v1.x, following properties are deprecated:

Property Replacement Comment
type="{success,warning,alert,error}" class="kwc-callout-{success,warning,alert,error}" We removed callout type and replace it by class names. The type did not change anything on the component behavior, only on its style. To avoid confusion and keep function, we just moved it. See demo/styles for more information.

Children selectors

Selector Description
* Content of the callout, can be anything. If there is no content, only the header part will be shown.

Methods

This component has no particular method.

Events

This component has no particular event (except notified attributes).

Custom styles

Name Default Description
--kwc-callout-header {} Styles for header part of callout, when existing
--kwc-callout-content {} Styles for content part of callout, when existing. This should stylizing only content container (eg: margin), not the content himself. If you want to style the content (eg: color), use kwc-callout {selector} instead.
